Manjaro Gnome 配置

2019-10-29

Contents

    • GFW

折腾过很多不同的桌面环境和不同的操作系统。 最值得折腾的其实是 i3 和 xmonad，很酷炫。有几个月甚至稳定用于生产。 直到后来我爱上了 Emacs 的快捷键。Tiling WM 与 Emacs 争抢 Alt 键使用权，搞得我自己也凌乱不堪。 最终返璞归真，Gnome 还是最符合日常使用的。嗯，还有 Windows。

软件源

首先替换为国内源：

1

sudo pacman-mirrors -GB testing -c China

等一段时间，会弹出一个小窗口，让你勾选最终要加入哪个。

然后，启动 AUR。直接操作图形界面：

启用以后，还要安装 yaourt:

1

pacman -S yaourt

GFW

Linux 下可以用的 SSR 软件是 electron-ssr。 用 yaourt 安装好以后，添加订阅即可。与其他平台下的 SSR 软件大同小异。

但我遇到了问题，这个软件貌似本身并不能控制 gnome 的 proxy 设置。 所以，只能自己再手动运行一次 genpac，生成代理文件，然后在 Settings -> Network 里面手动设置。

生成 pac 代理文件：

1
2
3
4
5
6
7
8
9

# 首先安装 genpac
sudo pip3 install -U pip
pip3 install genpac --user

# 生成文件
genpac --pac-proxy "SOCKS5 127.0.0.1:1080" \
    --gfwlist-proxy="SOCKS5 127.0.0.1:1080" \
    --gfwlist-url=https://raw.githubusercontent.com/gfwlist/gfwlist/master/gfwlist.txt \
    --output="autoproxy.pac"

然后，在 Settings -> Network -> Proxy 里面选择 Automatic，地址填入： file:///home/yourname/path/to/autoproxy.pac。

不过，有可能后续的版本已经解决了这个问题。我再次重启以后，手动设置的 pac 文件地址就不见了，变成了这样：

地址是： http://127.0.0.1:2333/proxy.pac

不管你是如何启动代理的，都可以通过右上方的 Electron 标志来选择”PAC代理”或者”全局代理”。管用的。 总之，这时你终于可以打开 Google 了。

最后，安装 proxychains4：

1

sudo pacman -S proxychains

然后配置 /etc/proxychains.conf。

输入法

曾经输入法里面唯一能打的就是搜狗输入法。 然而搜狗输入法在 linux 下问题实在太多了。 现在，gnome 自带的框架是 ibus，干脆放弃了搜狗，选择了次优的方案： Rime 输入法，又称小狼毫。

1

sudo pacman -S ibus-rime

然后进入 Settings -> Region & Language -> Input Sources，点击加号，添加 Rime 即可。

最后进入配置 Rime 的阶段。

建立文件 ~/.config/ibus/rime/default.custom.yaml:

1
2
3

patch:
    schema_list:
        - schema: luna_pinyin_simp

这样就变成了简体拼音。用以下命令重启一下 ibus-daemon 即可：

1

ibus-daemon -drx

如果希望后选词横向排列，再建立文件： ~/.config/ibus/rime/build/ibus_rime.yaml:

1
2

style:
    horizontal: true

以前的 ibus-setup 都不能用了。估计是要渐渐 deprecate 了。

外观

Gnome-Terminal 可以用 Gogh 来方便地添加 Theme：

1

yaourt -S gogh-git

字体用 Fira-Code：

1

sudo pacman -S ttf-fira-code

至于全局的 Theme，我觉得这个 Nordic Polar 就很不错。

其他

keyboard repeat rate 在 Settings -> Universal Access 里面。

Remap Capslock 的设置需要打开另一个软件 Tweaks，找 Keyboard & Mouse -> Keyboard -> Additional Layout Options.
