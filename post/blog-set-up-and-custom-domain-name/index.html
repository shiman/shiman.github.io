用Github做博客

2018-06-16

Contents

    • 域名设置

用Github Pages做博客已经是个很常见的事情了。 这里主要分享一下如何设置个人域名和HTTPS。

为什么用Github

科学上网是在国内的必备技能。 为了科学上网，不少人都有自己的VPS，用来搭建梯子。 比如说我。 而既然有了VPS，为什么还要用Github呢？ 还是因为干扰太多。 个人随便搞一个VPS，选的都是很便宜的套餐，没有备份，没有保护。 设置一个WordPress不难，但很难保证不会哪天被一堆垃圾评论爆掉， 变成最大线上赌场。 同时，国内的老大哥也不是吃素的，VPS如果不够低调，IP也分分钟被封。 而Github就基本不存在这两个问题了。 被微软收购的那一天，曾出现了破纪录的大规模DDOS攻击，都被企业级的防护措施挡住了。 至于老大哥这方面，目前只是速度慢而已，基本上没有封杀的情况发生。 当然也有例外哈，比如今年（18年）上半年的北大事件， 有人把事情原委用Github Issue的形式传播， 老大哥发现以后就把这个页面墙掉了。 不过这也仅仅是这一个页面而已，其他的不受影响。

关于如何建立一个Peronsal Github Page，这里就不细说了。 有很多静态博客框架，里面都提到了如何部署。

域名设置

我是从Godaddy买的域名，还算是划算吧。 从这里买域名不需要备案，比较省心。 具体的设置过程分为两步。

第一步 域名提供商DNS转发

首先，登陆Godaddy，账号管理，DNS设置页面。

以我自己的域名为例，shiman.me， 为了确保 shiman.me 和 www.shiman.me 都可以工作， 需要设置两种转发。

第一种，Type CNAME，对应于www域名，按照以下内容设置即可：

1
2
3
4

Type: CNAME
Name: @
Value: shiman.github.io
TTL: 1 Hour

如果已经有了Name为@的CNAME规则，直接在修改即可。 没有的话就创建一个。

第二种，Type A，对应于 apex 域名，不带www的。 这里需要参考一下官方说明。 官方会给出若干IP地址。 我设置的时候，官方说明给出了四个：

1
2
3
4

185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

为这四条地址逐一创建Type A转发规则即可。 比如第一条，内容如下：

1
2
3
4

Type: A
Name: @
Value: 185.199.108.153
TTL: 1 Hour

第二步 在Github上设置

这一步就很简单了。Github做了很好的集成。 打开你的Github Pages Repo， 比如github.com/yourname/yourname.github.io， 进入设置页面， 在最下方，”Danger Zone”的上面，有个”Custom domain”的面板。 填上自己的域名，如 www.shiman.me ，即可。 别忘了勾选下面的”Enable HTTPS”选项。 有时候这个多选框可能不可用，这种情况下等几分钟，再来看看。

域名设置的过程虽然不难，但各方响应不是很及时。 有任何问题，比如设置好了但是打不开， 或HTTPS页面提示证书不安全/不可用， 八成是还没有更新。 喝一杯咖啡，回来就好了。
